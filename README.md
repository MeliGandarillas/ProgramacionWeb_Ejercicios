# Programación Web. Unidad 2.

Este repositorio contiene la integración de 10 ejercicios prácticos desarrollados para la Actividad 1 del Tema 2, centralizados a través de un menú principal (index.html) y desplegados en línea mediante GitHub Pages.

## Objetivo.
* Integrar y consolidar los conocimientos fundamentales del desarrollo web front-end (HTML5, CSS3 y JavaScript básico), estructurando contenidos semánticos, aplicando estilos reutilizables y añadiendo lógica interactiva sin recarga de página.

## Tecnologías Utilizadas.
* HTML5: Para la estructura y semántica del contenido.
* CSS3: Para el diseño visual.
* JavaScript: Para la lógica matemática.

## Temas Implementados.
* Texto básico con etiquetas de Texto (`<h1>`) y párrafos (`<p>`), 
* Inserción de imágenes básicas y enlaces simples (URL)
* Listas anidadas ordenadas para la creación de menús con (`<ol>`) y no ordenadas (`<ul>`) combinadas con hipervínculos.
* Uso estricto de etiquetas semánticas (`<header>`, `<main>`, `<section>`, `<div>`) para garantizar la accesibilidad del sitio web.
* Diseño de una tabla compleja para representar un horario semanal. Se aplicó semántica avanzada dividiendo el contenedor con `<table>`, `<thead>`, `<tbody>`, y organizando las filas y celdas con `<tr>`, `<th>` y `<td>`.
* Caja de modelo (box-sizing: border-box) para el control exacto del espaciado interno (padding) y bordes para evitar desbordamientos en los elementos del formulario.
* Uso de CSS Grid combinando unidades flexibles (`1fr`, `rem`) para lograr una interfaz adaptable.
* Implementación de pseudoclases como (:hover) y (:focus) para mejorar la experiencia de usuario al interactuar con botones e inputs.
* Uso de atributos como (required) y (type="number") dentro de una etiqueta (`<form>`) para delegar la validación al navegador, optimizando el rendimiento.
* Manipulación del DOM con la captura de valores de entrada (.value) y renderizado de resultados.
* Control de eventos con ( e.preventDefault() ) en el evento submit para la operación matemática de celsius a fahrenheit en tiempo real, evitando el refresco de la página.
* Uso del atributo readonly en el campo de salida para que el usuario solo visualice y no pueda alterar el resultado manualmente.
